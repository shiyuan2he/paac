一>项目介绍
	web平台项目特点：技术新，技术范围广，分布式，高并发，集群，负载均衡，高可用，高吞吐量，海量数据（大数据），业务复杂，系统安全
	web平台运营模式：b2b：商家到商家－如阿里巴巴	b2c：商家到用户－如京东商城，天猫商城（b2b2c）	c2c:用户到用户－如闲鱼，转转	o2o:线上到线下	p2c:平台到用户
	paac（platform as a cloud－平台即是云）算是一个自我定义的p2c（平台到用户）的web项目。产品提供交流的平台，重点将关注用户。
二>技术选择
	传统性价比高的web 项目架构：lamj（linux + apache + mysql + java）	多虚拟机分布式,nginx负载均衡，redis缓存，solr搜索引擎
	分布式架构：1.把系统按照模块拆分成多个子系统	
			优点：a.把模块拆分，使用接口通信，降低模块之间的耦合度。
			      b.把项目拆分成多个子项目，便于多人分工合作
			      c.增加功能的时候只需要增加子项目，调用别的项目借口即可
			      d.可以灵活的进行分布式部署
			缺点：系统之间交互需要使用远程通信，借口开发增加工作量。
